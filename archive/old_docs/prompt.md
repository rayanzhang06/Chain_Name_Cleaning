创建一个医药连锁名称关联 agent，用于信息整理。
输入包括 1. KA专员客户关系数据模板【省份】.xlsx；2. 全国连锁简称.xlsx

KA专员客户关系数据模板【省份】.xlsx格式为：KA专员,KA专员ID,主数据代码,连锁全称,连锁简称,产品列表
全国连锁简称.xlsx 格式为：省份，连锁简称

用户故事：
1. 用户上传全国连锁简称.xlsx
2. 选择合适的 mcp 在线搜索各连锁简称，根据搜索结果，验证名称的有效性，给出置信度
3. 要求用户判断存疑简称的去留
4. 清洗后的全国连锁简称.xlsx存入数据库备用
5. 用户提供KA专员客户关系数据模板【省份】.xlsx，其中KA专员,KA专员ID,主数据代码,连锁全称,产品列表字段已填写完成。支持单个或批量清单处理
6. 询问该清单所属省份
7. 调用 LLM（kimi-k2-thinking-turbo），根据省份、连锁全称及第 2 步的搜索结果，寻找每个连锁全称的 1-3 个连锁简称备选，如无法找到备选简称则留空
8. 由用户决定最终的关联关系
9. 输出完整KA专员客户关系数据模板【省份】.xlsx

注意：
适当处理 xlsx 文件以避免大量 token 消耗。文件内容绝对不能改变！
